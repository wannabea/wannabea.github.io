Poj 1062 [最短路] + [枚举区间]

发表于 2019-10-06 分类于 解题报告 评论数： 阅读次数：

题目链接：http://poj.org/problem?id=1062

题意：

我们想要得到1这个人手里的物品（没错人就是物品吧…），可以直接花P元得到，也可以达成他的X个要求，得到Ti这个人手里的物品，再花Vi元的优惠价格得到1这个人的物品，我们要求得花费最少的钱得到1物品，但是有一个限制，每个人都有一个等级，我们交易的所有人的等级之差必须小于等于M。

输入方式：输入第一行是两个整数M，N（1 <= N <= 100），依次表示地位等级差距限制和物品的总数。接下来按照编号从小到大依次给出了N个物品的描述。每个物品的描述开头是三个非负整数P、L、X（X < N），依次表示该物品的价格、主人的地位等级和替代品总数。接下来X行每行包括两个整数T和V，分别表示替代品的编号和”优惠价格”。

题解

逆向思考，如样例，我们得到了第四个物品，就可以花低价得到第二个和第三个物品，得到第二个或者第三个，我们就可以低价获得第一个物品（前提是没有等级限制），这样，我们就可以从4到3和从4到2分别连一条有向边，边权分别为3和2对4的优惠价，从2到1从3到1分别连一条有向边，边权分别为1对2和3的优惠价。

但是我们不确定是它的优惠价格+另一物品价格便宜，还是直接购买它便宜，所以，我们构造一超级源点，指向每个物品连接有向边，边权为物品本身价格。

有向图构造完毕，接下来考虑等级问题。我们设超级源点的等级与1的等级相同，保证每次都可以跑。枚举所有存在的等级，每次跑dijkstra时判断是否符合区间就可以了。
